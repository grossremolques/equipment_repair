Funcionalidades

  1. Creación de solicitudes para reparación
    2. Características
    • Todos los campos son obligatorios
    • Según el tipo de reparación que elija, se habilitara o no el campo de Código Máquina/Equipo
    • En Código Máquina/Equipo sólo permitirá los items registrados en la lista de equipos
    • El campo Nombre equipo se autocompleta según el código
    • En Situación deberá hacer una breve descripción de la razón de la solicitud
    • En caso de la reparación referisrse a un equipo/maquinaria, el campo de Prioridad se autocompletará según información del listado de equipos, de lo contrario, usted deberá indicar el nive de prioridad.
    • En el campo Solicita, por defecto muestra el nombre del usuario que tiene la cuenta de correo asociada al inicio
  2. Diagnóstico
    3. Características
    • Se debe ingresar el id de reparación que será diagnósticado
    • De encontrar un id válido, cargará la información pertinente de la solicitud, y permitirá la carga de los demás datos
    • En el campo Diagnóstico se debe dar detalle del resultado de la revisión.
    • en el campo Atiende, por defecto muestra el nombre del usuario que tiene la cuenta de correo asociada al inicio
  3. Reparación
    4. Características
    • Se debe ingresar el id de reparación que será diagnósticado
    • De encontrar un id válido, cargará la información pertinente de la solicitud, y permitirá la carga de los demás datos
    • En el campo Acción se debe dar detalle de las acciones tomadas o las tareas realizadas para llevar a cabo la reparación.
    • Tambien se debe indicar las horas hombres utilizadas para la labor y la fecha de cuando se entregó el equipo a pañol/depósito
    • en el campo Responsable, por defecto muestra el nombre del usuario que tiene la cuenta de correo asociada al inicio
  4. Entrega
    5. Características
    • Se debe ingresar el id de reparación que será diagnósticado
    • De encontrar un id válido, cargará la información pertinente de la solicitud, y permitirá la carga de los demás datos
    • en el campo Responsable de entrega, por defecto muestra el nombre del usuario que tiene la cuenta de correo asociada al inicio

Tipos de Usuarios

  • Administrador: Tiene acceso a todas las funcionalidades del aplicativo web
  • Usuario A: Solo puede hacer solicitudes de reparación
  • Usuario B: Puede hacer Solicitudes, Diagnósticos y Reparaciones
  • Usuario C: Puede hacer Solicitudes y Entregas de pañol a otro sector
